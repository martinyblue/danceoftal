먼저 이 순서대로 누르세요

이 화면은 공식 dance-of-tal 화면이 아니라, 이 프로젝트를 관리하기 쉽게 만든 보조 화면입니다. 아래 버튼을 누르면 내 컴퓨터의 `.dance-of-tal/` 폴더에 실제 파일이 만들어집니다.

누르면 생기는 변화

1번
관리 폴더를 만들고 상단 상태가 준비 완료로 바뀝니다.
2번
사람 역할, 능력, 실행 에이전트, 업무 흐름 예시를 만듭니다.
3번
DOT Studio 화면에 기본 Performer와 Act를 보이도록 저장합니다.
4번
파일 목록, 미리보기, 그림 흐름을 최신 상태로 다시 읽습니다.

네 가지 부품 뜻

Tal
에이전트의 역할과 말투입니다.
Dance
반복해서 쓸 수 있는 능력 묶음입니다.
Performer
역할과 능력을 합친 실제 실행 에이전트입니다.
Act
여러 에이전트가 함께 일하는 순서입니다.

로그인 안내

GitHub Authorize 화면이 뜨면?

DOT Studio나 OpenCode에서 Sign in을 누르면 GitHub가 dance-of-tal 앱에 계정 확인 권한을 줄지 묻습니다. 지금 화면에 보이는 권한은 이메일 주소 읽기이며, registry publish/login 같은 기능을 쓸 때 필요합니다.

단순히 예시를 보고 로컬 파일을 만드는 단계라면 지금 승인하지 않아도 됩니다. 나중에 `dot publish`나 개인 registry 기능을 쓸 때 승인하세요.
