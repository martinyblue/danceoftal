Core Model

거대한 master prompt 대신, 에이전트 구성요소를 분리합니다.

Tal

누구인가

에이전트의 정체성, 성격, 기본 지시문을 담는 persona layer입니다.

Dance

무엇을 할 수 있는가

GitHub repo와 Agent Skill 형식으로 재사용되는 skill package입니다.

Performer

무엇이 실행되는가

Tal, Dance, model, MCP/tools 설정이 결합된 runnable agent입니다.

Act

어떻게 함께 움직이는가

여러 Performer의 관계, 규칙, 메시지 흐름을 명시하는 choreography입니다.

Why It Matters

subagent 호출을 재사용 가능한 participant 구조로 바꿉니다.

Before

main agent 중심

역할과 관계가 코드나 설정 파일에 숨어 있고, subagent는 일회성 내부 기능처럼 굳어지기 쉽습니다.

After

choreography 중심

각 agent를 독립 패키지로 유지하고, Act에서 누가 누구와 어떤 방향으로 협업하는지 명시합니다.

dot CLI

설치, 생성, 검색, 배포를 CLI 흐름으로 다룹니다.

dot init

프로젝트 안에 `.dance-of-tal/` workspace를 준비합니다.

dot add <owner>/<repo>

GitHub에서 Dance skill package를 가져와 로컬 bundle로 설치합니다.

dot create --kind performer

Tal, Dance, model, MCP/tools 설정을 묶어 실행 가능한 agent를 만듭니다.

dot publish

조합한 asset을 registry에 공유할 수 있는 형태로 발행합니다.

DOT Studio

Figma처럼 캔버스에서 agent workflow를 설계합니다.

DOT Studio는 Performer와 Act를 배치하고 연결하는 로컬 비주얼 편집기입니다. Tal과 Dance draft를 만들고, model과 MCP 설정을 조합하며, OpenCode가 실행할 runtime 정의로 projection합니다.
